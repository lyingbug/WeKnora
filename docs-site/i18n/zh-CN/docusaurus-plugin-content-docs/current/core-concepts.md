# 核心概念

## 租户与工作区

租户是成员、知识库、智能体、模型和审计日志的隔离工作区。RBAC 决定成员能查看或管理哪些资源。

租户通常用于隔离团队、业务线、客户或环境。租户边界一般也应该是权限边界。

## 知识库

知识库保存 WeKnora 可检索和用于回答的内容，可以包含文档、FAQ、URL 或生成的 Wiki 页面。

建议按知识领域和权限范围组织知识库。例如产品手册、客服话术和内部制度通常应该拆成不同知识库，避免无关内容影响检索质量。

## 知识条目

知识条目是知识库中的一个来源记录，例如 PDF、Markdown 页面、URL 导入或 FAQ。

知识条目会包含标题、来源、标签、状态和解析信息等元数据。这些元数据既方便管理，也能帮助检索系统过滤结果。

## 分块

分块是从知识条目中切分出的较小内容单元，用于检索和回答上下文组装。

分块质量会直接影响回答质量。分块太小容易丢上下文，分块太大又会降低检索相关性。WeKnora 支持按文档类型和模型上下文长度调整分块策略。

## RAG

RAG 指检索增强生成。WeKnora 先检索相关分块，再把上下文交给模型生成回答。

RAG 适合直接知识问答，尤其是需要回答有来源引用的场景。

## Agent

Agent 可以执行多步推理、调用工具、使用 MCP 服务、搜索知识库并生成最终答案。

Agent 模式适合需要规划、工具调用或多次查询的任务。它比普通 RAG 更灵活，但也更需要清晰的工具权限和可观测性。

## MCP 服务

MCP 服务向 Agent 暴露可调用工具。WeKnora 支持内置 MCP 服务和外部 MCP Server。

MCP 让 WeKnora 可以把 Agent 连接到外部能力，而不需要把每个工具都硬编码进核心系统。敏感工具可以通过审批流程保护。

## Agent Skill

Agent Skill 是一组可复用能力，可以包含指令、参考资料和可选脚本。重复性的专业工作流适合沉淀为 Skill。

## Wiki 模式

Wiki 模式把来源文档转化为结构化 Markdown 页面和可浏览的知识网络。

Wiki 模式和普通 RAG 不同。RAG 是在提问时检索分块，Wiki 模式会提前生成持久化的结构化知识层，用户可以浏览、链接、审阅和复用。

## 模型角色

WeKnora 会把不同模型用于不同任务：

| 角色 | 用途 |
| --- | --- |
| 对话模型 | 生成回答和执行 Agent 推理 |
| Embedding 模型 | 把文本转成向量，用于语义检索 |
| Rerank 模型 | 对召回候选进行精排 |
| VLM | 理解图片或视觉文档内容 |
| ASR | 把语音转成文本 |

## 检索组件

检索可以组合多种信号：

- 稠密向量检索：用于语义相似度。
- 稀疏或关键词检索：用于精确术语。
- 图谱增强检索：启用知识图谱时扩展关系。
- Rerank：优化最终上下文排序。
