# 文档入库

文档入库把来源内容转化为可检索、可用于回答的分块。

当你需要了解能导入什么、处理过程如何工作、以及如何排查入库失败时，可以从本页开始。

## 支持来源

- 本地文件上传
- URL 导入
- 手动 Markdown 录入
- 数据源同步
- FAQ 条目创建

## 支持格式

WeKnora 支持 PDF、Word、Markdown、文本、HTML、图片、CSV、Excel、PowerPoint 和 JSON 等常见格式。

具体解析效果会受部署配置影响，例如解析引擎、OCR、多模态能力是否启用等。

## 入库阶段

1. 上传或同步来源内容。
2. DocReader 提取文本和结构化内容。
3. 内容被切分为分块。
4. 生成 Embedding。
5. 保存分块和元数据。
6. 更新检索索引。

## 如何选择入库设置

不同内容应关注不同设置：

| 内容类型 | 建议关注 |
| --- | --- |
| 产品文档 | 标题结构、稳定分块、引用质量 |
| 复杂 PDF | 解析质量、表格抽取、页码引用 |
| 图片或扫描件 | OCR 和 VLM 配置 |
| 大型表格 | Sheet 结构、行分组、元数据 |
| FAQ | 问题归一化和标签 |
| Wiki 生成 | 图谱抽取和页面结构 |

## 重解析

以下情况适合重解析：

- 解析器设置变化。
- 分块策略变化。
- 配置了更合适的 Embedding 模型。
- 首次解析后启用了多模态或图谱抽取。
- 上一次解析失败或抽取文本质量较差。

重解析应保留原始来源，同时替换本次运行派生出的分块、Embedding 和索引。

## 用户应检查什么

入库后建议检查：

- 条目状态。
- 抽取文本预览。
- 分块列表和边界。
- 来源元数据和标签。
- 引用是否指向有用片段。

如果回答效果不好，第一步通常不是换模型，而是先检查抽取文本和分块质量。
