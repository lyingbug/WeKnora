# Agent 模式

Agent 模式适合需要多次检索、工具调用或规划的任务。

Agent 可以：

- 搜索一个或多个知识库。
- 调用 MCP 工具。
- 使用网络搜索。
- 在沙箱中运行经过批准的 Skill。
- 经过多步推理后生成最终回答。

## 何时使用

当任务需要规划、工具调用、跨来源推理或迭代查询时，使用 Agent 模式。

适合问题示例：

- “把这个知识库里的政策和最新公开信息做对比。”
- “找到相关内部流程，然后帮我起草回复。”
- “搜索知识库、调用一个 MCP 工具，然后总结结果。”

如果只是对已知知识库提直接问题，RAG 模式通常更快，也更容易审计。

## 配置 Agent

创建或编辑 Agent 时，应定义：

- Agent 的用途。
- 系统指令。
- 可搜索的知识库。
- 可调用的 MCP 工具。
- 是否启用网络搜索。
- 是否启用 Skills。
- 模型、超时和审批设置。

不要给一个 Agent 所有工具和所有知识库。范围更窄的 Agent 更容易理解，也更安全、更容易调试。

## 工具审批

敏感工具可以要求人工审批后再执行，适合外部调用、有副作用或需要安全控制的场景。

建议对以下工具启用审批：

- 使用私有凭据的工具。
- 会向第三方服务发送数据的工具。
- 会写入、删除或触发动作的工具。
- 成本较高或运行时间较长的工具。

## 查看 Agent Trace

调试 Agent 输出时，检查：

- 搜索了哪些知识库。
- 调用了哪些工具。
- 是否有工具调用卡在审批。
- 工具返回了什么观察结果。
- 使用了哪个模型。
- 最终回答是否引用了正确证据。
