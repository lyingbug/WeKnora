# Wiki 模式

Wiki 模式帮助团队把原始文档转化为结构化、相互链接、可浏览的知识。

当来源材料规模大、主题交织复杂，单纯问答不足以帮助用户理解全局时，可以使用 Wiki 模式。

## 产出内容

- Markdown Wiki 页面
- 相关概念之间的链接
- 可浏览的页面层级
- 知识图谱可视化

## 适用场景

当用户需要探索一组知识，而不是只问单个问题时，适合使用 Wiki 模式。

适合内容：

- 产品手册。
- 技术设计文档。
- 研究报告。
- 制度集合。
- 项目归档。
- 新人入门材料。

不太适合：

- 很小的 FAQ 集合。
- 不需要结构化的临时笔记。
- 不允许转换或摘要的内容。

## 典型流程

1. 创建 Wiki 知识库。
2. 上传或同步来源文档。
3. 让 WeKnora 生成 Wiki 页面。
4. 审阅页面和图谱关系。
5. 把 Wiki 作为检索和推理的结构化知识层。

## 审阅流程

生成页面也应像其他知识资产一样审阅。重点检查：

- 页面标题是否准确对应概念。
- 页面内容是否基于来源文档。
- 链接是否指向有用的相关页面。
- 重要来源章节是否遗漏。
- 图谱关系是否有意义。

## 和 RAG 的关系

Wiki 模式不替代 RAG。它会创建一个结构化知识层，后续 RAG 和 Agent 都可以复用。用户可以先浏览 Wiki 理解领域，再基于同一个知识库提问。
