# 模型配置

WeKnora 可以连接多种模型供应商和模型类型。

## 模型类型

- 用于回答生成和 Agent 推理的对话模型。
- 用于向量索引的 Embedding 模型。
- 用于检索精排的 Rerank 模型。
- 用于图片理解的 VLM 模型。
- 用于语音转写的 ASR 模型。

## 最小模型要求

大多数部署至少需要：

- 一个对话模型。
- 一个 Embedding 模型。

没有对话模型，WeKnora 无法生成回答。没有 Embedding 模型，语义检索和文档索引会受限。

## 可选模型角色

| 角色 | 何时启用 |
| --- | --- |
| Rerank | 相关分块能召回，但最终排序不好 |
| VLM | 文档包含图片、截图、扫描页或视觉内容 |
| ASR | 音频或语音内容需要入库 |
| 独立 Agent 模型 | Agent 推理需要不同的成本、延迟或能力配置 |

## 配置范围

模型可以按全局、租户或具体知识工作流配置，取决于部署方式和权限设置。

选择模型时需要综合考虑隐私、延迟、成本、上下文长度和语言能力。

## 选择检查清单

生产启用前，确认：

- API 端点和凭据。
- 模型名称和支持参数。
- 上下文长度。
- 是否支持流式输出。
- 限流和额度。
- 数据留存策略。
- 对文档语言的质量。
- 预期工作负载下的成本。

## 排障

模型调用失败时，应分别测试对话模型和 Embedding 模型。对话模型可用不代表 Embedding 配置正确，而 Embedding 失败会导致入库失败，即使对话仍能正常工作。
