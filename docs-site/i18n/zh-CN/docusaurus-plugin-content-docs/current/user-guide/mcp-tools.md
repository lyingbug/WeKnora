# MCP 工具

MCP 服务向 WeKnora Agent 暴露推理过程中可调用的工具。

MCP 是把 Agent 连接到外部能力的主要机制，不需要把每个集成都写进核心应用。

## MCP 服务类型

- 由 WeKnora 管理的内置 MCP 服务。
- 通过支持传输方式连接的外部 MCP Server。
- 团队自定义的 MCP 服务。

## 常见用途

- 查询业务系统。
- 获取外部数据。
- 执行安全自动化。
- 让 Agent 调用领域工具。

## 配置模型

一个 MCP 服务应定义：

- 服务名称和描述。
- 传输配置。
- 凭据或环境变量。
- 可用工具。
- 租户或 Agent 访问范围。
- 敏感工具的审批要求。

## 工具审批

如果工具可能暴露私有信息、调用外部系统、修改状态或产生成本，应启用审批。审批用户应能看到将要运行的工具和即将发送的参数。

对于访问外部系统或有副作用的工具，建议启用审批和最小权限凭据。

## 排查 MCP 工具

如果工具没有出现或无法运行，检查：

- MCP 服务是否启用。
- Agent 是否允许使用该服务。
- 凭据是否存在且有效。
- 后端是否能访问对应传输端点。
- 工具调用是否正在等待审批。
- 工具结果是否超过大小或格式限制。
