# 智能问答与 RAG

RAG 模式会先检索相关知识，再调用语言模型生成回答。

当用户需要基于已知来源获得有依据的回答时，优先使用 RAG 模式。大多数知识库问答都应该先从 RAG 开始。

## 请求流程

1. 用户提出问题。
2. WeKnora 搜索选定知识库。
3. 相关分块被排序并组装为上下文。
4. 模型生成回答。
5. 返回答案和引用来源。

## 适用场景

当问题可以直接通过知识库回答，并且来源依据很重要时，优先使用 RAG 模式。

合适问题示例：

- “部署指南里如何开启 Neo4j？”
- “总结上传文档里的退款政策。”
- “哪一节解释了 API Key 认证？”

如果任务需要多次工具调用、规划、网络搜索或跨系统操作，应使用 Agent 模式。

## 来源引用

引用是可信回答的一部分。好的回答应让用户能检查信息来自哪里。

如果引用缺失或质量差，检查：

- 选定知识库是否已经生成分块。
- 检索是否返回相关分块。
- 回答是否基于上下文生成，而不是模型常识。
- Prompt 或回答模式是否抑制了引用输出。

## 提升质量

- 保持知识库主题聚焦。
- 使用清晰的文档标题和元数据。
- 配置合适的 Embedding 模型。
- 候选内容较多时启用 Rerank。
- 回答不完整时检查引用分块。

## 常见调优路径

1. 确认文档解析质量。
2. 检查分块边界。
3. 用代表性问题测试检索。
4. 如果精确术语召回差，增加关键词或混合检索。
5. 如果召回到了但排序差，增加 Rerank。
6. 将无关内容拆到不同知识库。
